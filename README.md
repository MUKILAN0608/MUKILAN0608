<details>
<summary><b>snake grid not showing? read this</b> — the two most common causes</summary>
<br>

The snake SVG only exists once a GitHub Action has actually run and pushed it to an `output` branch. A blank/broken image almost always means one of these:

1. **The workflow never ran.** Adding the YAML below to your repo does not run it automatically the first time — go to **Actions → snake → Run workflow** and trigger it manually once.
2. **Workflow permissions are read-only.** Go to **Settings → Actions → General → Workflow permissions** and set it to **Read and write permissions**, otherwise the push step fails silently.
3. **Repo name mismatch.** This only works in your special profile repo, named exactly `MUKILAN0608/MUKILAN0608`.
4. **The `output` branch doesn't exist yet.** After a successful run, check the branch dropdown — you should see `main` and `output`. If `output` is missing, the push step failed (see #2).

```yaml
name: snake

on:
  schedule:
    - cron: "0 */6 * * *"
  workflow_dispatch:
  push:
    branches: [ main ]

permissions:
  contents: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: generate snake svg
        uses: Platane/snk@v3.5.0
        with:
          github_user_name: MUKILAN0608
          outputs: |
            dist/github-contribution-grid-snake.svg?color_snake=%23C1121F&color_dots=%230D1117,%238B0000,%23C1121F,%23FF3B3B,%23E6EDF3
            dist/github-contribution-grid-snake-dark.svg?palette=github-dark&color_snake=%23FF3B3B&color_dots=%230D1117,%238B0000,%23C1121F,%23FF3B3B,%23E6EDF3

      - name: push to output branch
        uses: crazy-max/ghaction-github-pages@v4
        with:
          target_branch: output
          build_dir: dist
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

The stats, streak, top-languages, activity graph, and calendar above don't need any of this — they render live from the API with zero setup, so the page never looks empty even before the snake workflow is running.
